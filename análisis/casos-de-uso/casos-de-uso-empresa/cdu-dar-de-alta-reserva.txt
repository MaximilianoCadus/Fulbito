Caso de uso: Dar de alta reserva

Actores:
- Usuario empresa (Primario)

Pre-condiciones:
- Se debe tener acceso a internet
- Se debe tener una sesión iniciada

Camino básico:
1- El usuario ingresa a la sección de reservas
2- El sistema muestra las reservas realizadas
3- El usuario selecciona dar de alta nueva reserva
4- El sistema muestra los datos necesarios para la nueva reserva
5- El usuario ingresa los datos de la reserva
6- El sistema valida la reserva y muestra mensaje de éxito

Caminos alternativos:
6.1- La reserva ingresada es inválida
6.1.1- El sistema no logra validar la reserva y muestra mensaje de error

Escenario de fracaso:
- El usuario no logra dar de alta la reserva

Escenario de éxito:
- El usuario logra dar de alta la reserva
