Funcionales
- El sistema debe permitir registrar usuarios jugador
- El sistema debe permitir registrar usuarios empresa
- El sistema debe premitir reservar canchas de fútbol
- El sistema debe permitir editar el perfil del usuario
- El sistema debe mostrar los distintos predios
- El sistema debe mostrar las distintas canchas de fútbol pertenecientes a un predio
- El sistema debe ofrecer Mercado Pago como medio de pago
- El sistema debe permitir buscar canchas según cantidad de jugadores y tipo de piso
- El sistema debe permitir cancelar una reserva con anticipación mayor a 24hs
- El sistema debe enviar un correo al usuario jugador que realizó la reserva con los detalles de la misma
- El sistema debe enviar un correo al usuario empresa la reserva de un usuario jugador con los detalles de la misma
- El sistema debe enviar un correo al usuario jugador pidiendo una valoración sobre el servicio
- El sistema debe enviar un correo mensual al usuario empresa pidiendo una valoración sobre el servicio

No funcionales
- El sistema debe tener un tiempo de respuesta óptimo de 1 segundo
- El sistema debe tener una interfaz intuitiva

De dominio
- Los usuarios pertenecen a una de dos categorías (jugador o empresa)
- Los usuarios jugador van a poder reservar canchas de fútbol
- Los usuarios empresa van a poder publicar sus predios y canchas de fútbol
