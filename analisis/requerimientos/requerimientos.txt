Funcionales
- El sistema debe registrar jugadores
- El sistema debe registrar empresas
- El sistema debe registrar predios asociados a empresas
- El sistema debe registrar canchas asociadas a predios
- El sistema debe permitir modificar el perfil de los tres tipos de usuarios: empresa, predio, jugador
- El sistema debe permitir modificar la información de la cancha
- El sistema debe mostrar canchas acorde a parámetros de búsqueda
- El sistema debe permitir reservar canchas
- El sistema debe permitir cancelar reservas
- El sistema debe permitir subir comprobante del pago de la seña de la reserva
- El sistema debe enviar un correos al jugador y al predio cuando se realiza una reserva o cuando cambia el estado de la misma
- El sistema debe mostrar reportes por predio de las reservas y el dinero

No funcionales
- El sistema debe tener un tiempo de respuesta óptimo de 1 segundo
- El sistema debe tener una interfaz intuitiva

De dominio
- Los usuarios pertenecen a una de tres categorías (jugador - empresa - predio)
- Los jugadores van a poder reservar canchas de fútbol
- Las empresas van a poder publicar sus predios y canchas de fútbol
