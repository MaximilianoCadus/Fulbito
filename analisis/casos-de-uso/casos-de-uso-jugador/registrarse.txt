Caso de uso: Registrarse

Actores:
- Jugador (Primario)

Pre-condiciones:
- Se debe tener acceso a internet

Camino básico:
1- El usuario ingresa su mail, contraseña y datos requeridos para registrarse
2- El sistema valida los datos ingresados por el usuario y muestra mensaje de éxito

Caminos alternativos:
2.1- Los datos ingresados son inválidos
2.1.1- El sistema no logra validar los datos ingresados por el usuario y muestra mensaje de error

Escenario de fracaso:
- El usuario no logra registrarse ya que los datos ingresados son inválidos

Escenario de éxito:
- El usuario logra registrarse
