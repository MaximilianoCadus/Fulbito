Caso de uso: Dar de alta predio

Actores:
- Empresa (Primario)

Pre-condiciones:
- Se debe tener acceso a internet
- Se debe tener una sesión iniciada

Camino básico:
1- El usuario ingresa a la sección de predios
2- El sistema muestra los predios
3- El usuario selecciona dar de alta nuevo predio
4- El sistema muestra los datos necesarios para el nuevo predio
5- El usuario ingresa los datos del predio
6- El sistema valida el predio y muestra mensaje de éxito

Caminos alternativos:
6.1- El predio ingresado es inválido
6.1.1- El sistema no logra validar el predio y muestra mensaje de error


Escenario de fracaso:
- El usuario no logra dar de alta un predio ya que los datos ingresados son inválidos

Escenario de éxito:
- El usuario logra dar de alta un predio
